# ADR-027: Sales Anchor UI の i18n 対応（日本語/英語切り替え）

| 項目 | 内容 |
|------|------|
| ステータス | Accepted |
| 作成日 | 2026-05-14 |
| 関連 | ADR-012, ADR-019 |

## What

Sales Anchor の UI を多言語対応にし、ユーザーが言語を切り替えられるようにする。

- 対応言語: 日本語、英語（初期リリース）
- 対応範囲:
  - すべての UI ラベル（ボタン、見出し、フォームラベル、メニュー）
  - ドロップダウン選択肢（ステータス、温度感、流入元、タイプ等）
  - エラーメッセージ・通知メッセージ
  - 日付・通貨フォーマット
- 切り替え方法: ユーザーごとの言語設定（プロファイル画面で選択可能）
- 永続化: tenant_users テーブルに locale カラム追加（または既存設定テーブル拡張）

## Why

### グローバル展開の前提整備

Sales Anchor は B2B 輸出 CRM として海外顧客とのコミュニケーション機能を提供している。営業担当者本人が日本語ユーザーである一方、運用業務の中で英語環境が必要な場面（海外オフィス、英語話者のメンバー追加等）が想定される。

### 技術的負債の解消

現状の UI は日本語ラベルがコード内にハードコードされており、以下の問題がある:

- 翻訳・修正のために実装変更が必要
- A/B テストや表現変更のコストが高い
- グローバル展開時に大規模リファクタリングが発生

パートナー Claude Code が過去に「日本語をハードコードせずに切り替えできるデータベースにしたい」と提案していた経緯があり、本 ADR でその方針を正式採択する。

### Meta App Review 等の海外向け要件への対応

Meta App Review 公式ガイドラインは「English を推奨」としており、現状は日本語 UI + 字幕で対応するが、本格的な i18n 対応により将来的なレビュー・申請プロセスを簡略化できる。

### B2B SaaS の標準機能

競合 SaaS（Salesforce、HubSpot 等）は標準で多言語対応しており、エンタープライズ顧客の獲得には必須機能となる。

## Scope 外

- 撮影本番への適用（撮影は日本語 UI + 字幕で対応）
- 第三言語（中国語、スペイン語等）の追加（将来 ADR で別途対応）
- ユーザー入力データ（顧客名、会社名等）の翻訳機能
- メール・通知の多言語化（別 ADR で対応）
- 技術選定の詳細（i18n ライブラリ、DB スキーマ設計はパートナー Claude Code の技術判断）

## 事業上の制約

### 優先順位

Meta App Review 申請完了後に着手する。撮影スケジュール優先のため、即時実装は行わない。

### リリース戦略

- フェーズ 1: 全 UI ラベルの i18n 化（日本語/英語切り替え可能）
- フェーズ 2: ユーザー設定 UI の追加（プロファイル画面で切り替え）
- フェーズ 3: ドキュメント・通知の多言語化

各フェーズは独立した PR / ADR で実装する。

### 既存ユーザーへの影響

デフォルト言語は日本語に維持し、既存ユーザーの体験を変更しない。明示的に英語を選択したユーザーのみ英語表示となる。

### ADR-012 ワークフロー準拠

技術判断（i18n ライブラリ、データベース設計、移行戦略）はパートナー Claude Code に委ねる。Web Claude は方針定義（What / Why / Scope 外）のみ起案する。

## 受け入れ基準（フェーズ 1）

1. ユーザーが UI から言語を「日本語」「英語」のいずれかに切り替えられる
2. 切り替え後、すべての UI ラベル・ドロップダウン選択肢・メッセージが選択言語で表示される
3. 設定は永続化され、再ログイン後も保持される
4. 既存ユーザーのデフォルト体験（日本語）が変更されない
5. リード等の業務データには影響しない
