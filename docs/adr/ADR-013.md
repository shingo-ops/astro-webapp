# ADR-013: ヘッダーロゴと favicon を Sales Anchor 正式ブランドに統一

## ステータス
提案

## What
1. アプリ上部ヘッダーに表示されている「🔗 CRM」絵文字＋テキストを、Sales Anchor 正式ロゴに置き換える
2. ブラウザの favicon を、Sales Anchor 正式アイコンに置き換える
3. iOS ホーム画面追加時の Apple Touch Icon を Sales Anchor 正式アイコンに設定する
4. URL 共有時の OG 画像を Sales Anchor 正式画像に設定する

## Why
- Meta App Review 用スクリーンキャスト録画の直前であり、ヘッダーロゴと favicon の未統一が録画ブロッカーになっている
- PR #195（feature/morimoto/branding-sales-anchor）でブランド表記は統一済だが、ヘッダーロゴと favicon の差し替えが漏れていた
- 審査担当者および将来のユーザーに対して、ブランドアイデンティティが整合した状態で提示する必要がある

## Scope外
- ロゴデザインの新規作成（PO提供の既存アセットを採用）
- ダークモード／テーマ機能の改修（録画前の設定変更で解決、`docs/dark_mode_3value_phase2_proposal.md` で Phase 2 投入予定として記録済）
- LP（salesanchor.jp）、Discord通知、メールテンプレートなど frontend アプリ以外のロゴ
- ヘッダー・favicon・apple-touch-icon・og-image 以外の UI 変更
- 既存の認証フロー、ナビゲーション構造、スタイリングシステムへの変更
- PWA 化（manifest.json の整備等）

## 事業上の制約
- Meta App Review スケジュール優先：本ADR施行は録画当日中に完了
- 既存ユーザー（HIGH LIFE JPN 本番テナント）への移行期間不要、即時切替で問題なし
- ロゴアセットは本ADRと同コミットで `frontend/public/` 配下に配置済（透過PNG形式）：
  - `favicon.png` (1024×1024) — favicon マスター
  - `logo.png` (1200×300) — ヘッダー用ロゴ
  - `apple-touch-icon.png` (180×180) — iOS用
  - `og-image.png` (1200×630) — URL共有プレビュー用
- 実装側は上記アセットを参照し、index.html の `<link>` タグおよびヘッダーコンポーネントを更新する
