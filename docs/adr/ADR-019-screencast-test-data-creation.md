# ADR-019: Meta App Review 撮影用テストデータの作成方針

| 項目 | 内容 |
|------|------|
| ステータス | Proposed |
| 作成日 | 2026-05-14 |
| 関連 | ADR-012, ADR-016, ADR-018, ADR-025 |

## What

Meta App Review 撮影本番に向けて、Sales Anchor の Leads 一覧画面に表示するテストリードを以下の方針で準備する。

- 件数: 7 件（既存の LD-00001, LD-00002 と合わせて計 9 件）
- 作成手段: Sales Anchor の Leads 新規登録 UI から Shingo が手動で作成
- データ内容: 完全に架空の顧客情報（実顧客データの混入を厳格に排除）
- メールドメイン: `example.com`（RFC 2606 予約ドメイン、実在しないことが保証されている）
- 電話番号: 入力しない（揺れ回避）
- 多様性: 国籍・ステータス・温度感を分散させ、業務リアリティを表現

## Why

### 撮影上の必要性

Meta App Review のレビュアーは、撮影された動画のみで申請内容を判断する。Dashboard が空の状態だと「業務として運用されている SaaS である」説得力を欠き、申請却下リスクが高まる。Meta 公式ガイドライン「A screencast must explain the app like the reviewer has never seen it before」に基づき、業務的に自然な状態を再現する必要がある。

### プライバシー保護の必要性

実顧客のデータを撮影で映すと、データ保護違反（GDPR 等）および Meta 審査での rejection リスクが発生する。架空データのみを使用することで、これらのリスクを排除する。

### ADR-025 との整合性

ADR-025 は「OAuth フロー / アプリケーションロジック外でのデータ DB 直接操作」を禁止しているが、本 ADR で定める作成手段は **Sales Anchor の正規 UI（Leads 新規登録画面）** 経由であり、ADR-025 の対象外。アプリケーションロジック経由の正規データ作成として完全に整合する。

## Scope 外

- DB 直接 INSERT による作成（ADR-025 に基づき禁止）
- 実顧客データの使用
- 撮影後のテストデータ自動クリーンアップ（手動削除で対応）
- tenant_meta_config 等の接続状態テーブルへの操作
- 撮影本番の手順そのもの（別途 docs/META_APP_REVIEW_SCREENCAST_SCRIPT.md で定義）
- AI 音声生成・通知欄クリア等の撮影手順（手作業準備として ADR 対象外）

## 事業上の制約

### 撮影スケジュール

Meta App Review 提出に向けた撮影本番を数日以内に予定。本データ作成は撮影開始の前提条件。

### 撮影後の取り扱い

撮影完了後、テストリードは撮影用途を終えるため、以下のいずれかを選択：
- 削除する（meta_messages の過去テストデータ削除と同様、UI または管理者操作で実施）
- 残す（次回撮影や運用テスト用途で活用）

判断は撮影完了後に Shingo が行う。

### 申請承認後

Meta App Review 通過後の Live モードでは、本テストリードは Production 環境の表示要件には不要となる。実顧客の登録開始タイミングで整理する。

## 受け入れ基準

1. Leads 一覧画面に 9 件（既存 2 + 新規 7）のリードが表示される
2. 全 7 件の新規リードのメールアドレスが `example.com` ドメインである
3. 顧客名・会社名・メールがすべて架空であり、実在の人物・企業と一致しない
4. tenant_004 内に作成されており、他テナントへの影響がない
