# ADR-019: Meta審査期間中の英語UI一時デプロイ

## Status
Accepted — 2026-05-09（実装手順は [docs/runbooks/adr-019-english-ui-temporary-deploy.md](../runbooks/adr-019-english-ui-temporary-deploy.md)）

## What
Meta App Review 期間中のみ、`recording/english-ui` ブランチを
本番環境（app.salesanchor.jp）に一時デプロイする。
審査通過後は develop ブランチの日本語 UI に戻す。

## Why
Meta のレビュアーはスクリーンキャストだけでなくアプリを LIVE で
操作して Permission の動作を確認する。
本番アプリが日本語 UI のままだとレビュアーが操作できず
リジェクトリスクが高まる。

## Scope外
- i18n 基盤の整備（react-i18next 等の導入）
- 永続的な英語 UI 化
- バックエンド API のレスポンス英語化

## パートナーへの委任事項
1. `recording/english-ui` を main にマージしてデプロイする手順の実装
2. 審査通過後に develop を main に再マージして日本語 UI に戻す手順の整備
3. `deploy.yml` への影響確認（migration046 等の冪等性確認）

## 実装結果

上記 3 項目はすべて [docs/runbooks/adr-019-english-ui-temporary-deploy.md](../runbooks/adr-019-english-ui-temporary-deploy.md) に統合実装済み:

- §2 — `recording/english-ui` を最新 `main` に追従させた上で PR 経由マージするデプロイ手順
- §4 — Meta 審査通過後に `develop → main` PR で日本語 UI に戻すロールバック手順
- §3 — `deploy.yml` 経由で走る全 migration（013〜046）の冪等性ガードを精査。migration046 は `ADD COLUMN IF NOT EXISTS` / `CREATE TABLE IF NOT EXISTS` / `CREATE OR REPLACE FUNCTION` / RLS ポリシーの存在チェック等で完全 idempotent と確認。**deploy.yml への改修は不要**

[docs/handoff/meta-screencast-handoff-2026-05-09.md](../handoff/meta-screencast-handoff-2026-05-09.md) §5 の「`recording/english-ui` は merge / 本番反映なし」記述は本 ADR で上書きされる（同日付の後発判断）。
