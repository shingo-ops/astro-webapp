# ADR-033: アプリ内テーマ切り替え（ライト / ダーク）

| 項目 | 内容 |
|------|------|
| ステータス | Proposed |
| 作成日 | 2026-05-14 |
| 関連 | ADR-027（i18n）, ADR-030（ログイン画面）|

## What

ユーザーがアプリ内でライト / ダークテーマを切り替えできる機能を実装する。

- ヘッダーに ☀️ / 🌙 テーマ切り替えボタンを追加（🌐 言語切り替えの隣）
- `users.theme` カラム追加（`'light'` / `'dark'`、DEFAULT `'light'`）
- クリックで全画面即時切り替え、DB に永続化
- OS の `prefers-color-scheme` 設定を**無視**し、アプリ設定を優先
- 全ユーザーのデフォルトは `'light'`（ライトモード）
- review@salesanchor.jp のテーマも `'light'` に初期設定
- ADR-027（i18n）と同じアーキテクチャパターンで実装

## Why

### OS 設定依存の解消

ADR-030 でログイン画面に `var(--bg-primary)` を採用したが、この変数は OS の `prefers-color-scheme` に依存しているため、ユーザーの Mac がダークモードだとネイビー背景になる。撮影時に OS 設定を変更する必要があり、ユーザー体験として不自然。

### ユーザー個人の設定として保持

ADR-027 の言語切り替えと同様に、テーマもユーザー個人の好みとして DB に保持することで、デバイスやブラウザが変わっても設定が引き継がれる。

### 撮影環境の統一

Meta App Review 撮影時に review@salesanchor.jp のテーマを 'light' に固定することで、OS 設定に依存しないクリーンな撮影環境を保証できる。

## Scope 外

- OS テーマの自動追従（`prefers-color-scheme` の利用は廃止）
- 組織 / テナント単位のテーマ強制設定
- カスタムカラーテーマ（ライト / ダークの 2 択のみ）
- ダークモードの完全な色調整（既存の CSS 変数を活用、細かい調整は別 ADR）
- ログイン画面のテーマ切り替え（ログイン後のみ対応）

## 事業上の制約

- 全ユーザーのデフォルトは `'light'`（既存ユーザーへの影響最小化）
- review@salesanchor.jp のテーマは migration で `'light'` に設定
- migration 054 として deploy.yml に登録（ADR-027 の migration 053 と同じパターン）
- ADR-027（i18n）の `users.locale` と同様に `users.theme` カラムとして実装
- `PATCH /staff/me/theme` エンドポイントを追加（locale と同じパターン）
- マージ判断は Shingo（自動マージ禁止）

## 受け入れ基準

1. ヘッダーに ☀️ / 🌙 ボタンが全画面で表示される
2. クリックでライト / ダーク即時切り替え
3. リロード後も設定が維持される（cookie + DB）
4. OS 設定（ダークモード）に関わらず、アプリ設定が優先される
5. 全ユーザーのデフォルトが `'light'`
6. review@salesanchor.jp のテーマが `'light'`
7. migration 054 が deploy.yml に登録済み
8. 既存テスト全てパス
