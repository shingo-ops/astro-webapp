# PROPOSAL-001: 開発フロー再設計（役割分担モデル）

## ステータス

**採択済み — ADR-012 として正式化（2026-05-07）**

> 本提案の内容は `docs/adr/ADR-012-what-how-separation.md` として ADR 化・採択された。
> 運用上の参照先は ADR-012 を使用すること。

## 概要

Shingo（PO）/ Web Claude / パートナー Claude Code の3者間で、
What（何を実現するか）と How（どう実装するか）を明確に分離した開発フローを採用する提案。

ADR-011 の3者協調モデルをベースに、パートナー Claude Code を
「指示通りに動くワーカー」ではなく「シニアエンジニアとしての判断ができる主体」として扱う。

## 経緯

- ADR-011 の自動レビューパイプラインに終了条件がなく、Revision 無限ループリスクが顕在化
- パートナー Claude Code が ADR 指示なしに自発的にセキュリティ強化（RLS、Zero Trust、静的解析 CI 等）を実装したことが判明
- このエビデンスをもとに、詳細な実装指示を ADR に書くことは過剰設計と判断
