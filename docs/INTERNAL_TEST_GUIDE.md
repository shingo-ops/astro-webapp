## 0. このガイドの目的

Sales Anchor を一般顧客に公開する前に、実業務を模した使い方で**使い勝手・バグ・機能不足**を洗い出すことが目的です。

**テスターの皆さんにお願いしたいこと**:
- 普段の業務を想像しながら操作する
- 「ここが分かりにくい」「これができない」と感じたら必ず記録する
- 不具合を見つけたら再現手順と一緒に報告する

---

## 1. ログイン方法

### 1-1. アクセス

1. ブラウザ（Chrome / Safari / Edge 推奨）で https://jarvis-claude.uk を開く
2. ログイン画面が表示される

### 1-2. 認証情報

- **メールアドレス**: 各テスターに個別発行（別途連絡）
- **初回パスワード**: 別途連絡
- **初回ログイン後**: 必ず各自でパスワード変更してください

### 1-3. ログインできない時

| 症状 | 対処 |
|------|------|
| 「メール or パスワードが違う」 | 大文字小文字・全角半角を再確認 |
| 「アカウントが存在しない」 | しんごさんに連絡 |
| ページが真っ白 | ブラウザのキャッシュクリア→再読み込み |
| それでも駄目 | スクリーンショット + 発生時刻をメモし報告 |

## 2. 主な機能と試してほしいシナリオ

### 2-1. 顧客管理

```
想定シナリオ:
1. 新規顧客を3社登録する（業種・規模をバラす）
2. 登録した顧客を検索・絞り込みする
3. 1社の情報を編集する
4. CSVでエクスポートしてExcelで開く
```

**見てほしいポイント**:
- 必須項目が足りているか？多すぎないか？
- 検索はイメージ通りに動くか？
- CSVの文字コード（日本語）は崩れていないか？

### 2-2. 商談（案件）管理

```
想定シナリオ:
1. 2-1 で登録した顧客に紐づけて商談を作成
2. ステージを進捗させる（例: 初回訪問→提案→受注）
3. 金額・期日を編集
4. ダッシュボードで金額の集計を確認
```

**見てほしいポイント**:
- ステージの名前・順序は業務実態と合うか？
- 金額の合計・グラフは正しく見えるか？

### 2-3. 注文管理

```
想定シナリオ:
1. 受注した商談から注文に変換
2. 数量・単価を入力
3. 合計金額が自動計算されているか確認
```

**見てほしいポイント**:
- 消費税の扱いは想定通りか？
- 異常値（0円、マイナス、巨大数字）を入れた時の挙動

### 2-4. ダッシュボード

```
想定シナリオ:
1. TOP画面で全体状況を俯瞰
2. 期間フィルタを切り替える
3. 数字をクリックして明細に飛べるか確認
```

**見てほしいポイント**:
- 朝一で開いた時に欲しい情報が揃っているか？
- 読み込み速度は待てる範囲か？

## 3. セキュリティ・注意事項

### 🚫 やってはいけないこと

- 他人のアカウントでログインしない
- パスワードをメール・チャットでそのまま送らない
- 個人の PC 以外（ネットカフェ等）で使わない
- テスト中に気づいた脆弱性を**テスト期間外の人に口外しない**

### ⚠️ 投入するデータについて

- **本物の顧客情報・個人情報は絶対に入れない**
- 社名は「テスト株式会社A」「サンプル商事」等の架空名を使用
- 電話番号は 000-0000-0000 等のダミーでOK
- メアドは test1@example.com 等のダミーでOK

## 4. トラブル時の連絡

### 緊急度の判断

| 緊急度 | 症状 | 連絡方法 |
|--------|------|---------|
| 🔴 高 | 画面が一切開かない、データが消えた | 即電話 |
| 🟡 中 | 特定機能だけ動かない、エラー表示 | Slack / メール |
| 🟢 低 | UIの違和感、要望 | フィードバックシートに記入 |

### 報告に必要な情報

1. **発生時刻**（年月日 時:分）
2. **操作手順**（何を押したら何が起きた）
3. **スクリーンショット**（可能なら）
4. **ブラウザ名・バージョン**（例: Chrome 129）
5. **OS**（Mac / Windows）
