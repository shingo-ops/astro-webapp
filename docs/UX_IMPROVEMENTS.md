# UX改善課題リスト

内部テスト中に発見されたUX観点の改善項目。内部テスト完了後にまとめて対応する。

## フォーム入力系

### 1. 顧客作成フォーム: 電話番号の入力ガイドがない
- **現象**: ユーザーが `1234` のような不正値を入力→サブミット→422で弾かれる
- **期待**: フロント側で以下いずれかの対策
  - `placeholder="例: 03-1234-5678 または 090-1234-5678"` を表示
  - blurイベントでクライアントサイドバリデーション（同じ正規表現を使用）
  - サブミット前に形式チェック、NGなら送信せずインラインエラー表示
- **関連バリデータ**: `backend/app/schemas/base.py:validate_phone`（正規表現 `^(\+?\d{10,15}|0\d{9,10})$`、ハイフン・スペース・括弧は自動除去）
- **発見日**: 2026-04-14（D-13 E2Eテスト中、hitoshi）
- **該当画面**: 顧客管理 → 新規作成

### 2. ダッシュボード: 「最近の商談」でステータスが英語(enum値)のまま表示
- **現象**: ダッシュボード下部の「最近の商談」テーブルで `status` 列に `negotiating` などの英語Enum値がそのまま表示される
- **期待**: `DealsPage` 同様に `STATUS_LABELS` で日本語化（`交渉中`など）、さらに `.badge-{status}` クラスで色付け
- **関連コード**: `frontend/src/pages/DashboardPage.tsx`（`DealsPage.tsx:19-21` の `STATUS_LABELS` を共通化すると良い）
- **発見日**: 2026-04-14
- **該当画面**: ダッシュボード → 最近の商談

### 3. ダッシュボード: 「進行中の商談」KPIのラベルが誤解を招く
- **現象**: 「進行中の商談」は内部的に `status='open'` のみカウント。ユーザーは `negotiating`（交渉中）や `on_hold`（保留）も"進行中"と捉えがちで、商談を作ったのに0と表示されて混乱
- **期待**: 以下いずれか
  - ラベルを「未着手の商談」や「オープンな商談」に変更して意味を明確化
  - または集計範囲を `open / negotiating / on_hold` まで広げる（ビジネス要件次第）
  - 「成約商談」「成約金額」も `won` のみ集計されることを明示（ツールチップなど）
- **関連コード**: `backend/app/routers/dashboard.py:73-76`、`frontend/src/pages/DashboardPage.tsx` のラベル
- **発見日**: 2026-04-14
- **該当画面**: ダッシュボード上部KPI

### 4. 削除ボタンの確認ダイアログがブラウザ標準で気付きにくい
- **現象**: 顧客/商談/注文の削除には `window.confirm()` が実装済みだが、ブラウザネイティブのシンプルなダイアログのため「確認した感」が薄く、誤クリックで通過しがち
- **期待**: アプリ内モーダル（モーダル overlay + 「本当に削除しますか？」+ 削除内容のサマリ表示 + 赤色の「削除する」ボタン）に置き換え
- **関連コード**:
  - `frontend/src/pages/CustomersPage.tsx:78`
  - `frontend/src/pages/DealsPage.tsx:96`
  - `frontend/src/pages/OrdersPage.tsx:96`
  - 共通モーダルコンポーネント化が望ましい
- **発見日**: 2026-04-14
- **該当画面**: 顧客管理 / 案件管理 / 注文管理 すべての削除ボタン

### 5. ログアウトボタン: 確認ダイアログがなく即実行される
- **現象**: ログアウトボタンを押すと即座にログアウトされ、ログイン画面に遷移する
- **期待**: 「ログアウトしますか？」の確認ダイアログを挟む（誤クリック防止、入力中データ保護）
- **関連コード**: ログアウトハンドラ（`frontend/src/` 内、Layout/Header系コンポーネント想定）。`window.confirm("ログアウトしますか？")` 程度の簡易実装でもOK
- **発見日**: 2026-04-14
- **該当画面**: 全画面共通（ヘッダー/メニューのログアウトボタン）
