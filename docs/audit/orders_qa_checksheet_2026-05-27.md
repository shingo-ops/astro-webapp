## 既知制約（Known Limitations）

QA 中にこれらが観測されてもバグ扱いしない（備考欄に `KL-ID` を書き、`N/A (KL-xxx)` で記入）。

| ID | 制約 | 影響範囲 | 根拠 |
|----|------|----------|------|
| **KL-F1** | 受注一覧ページの N+1 取得（売上 / 発送 / 仕入 / 報酬を行ごとに別 API 呼び出し） | 受注一覧の体感遅延 (件数が増えると顕著) | Sprint 5 Reviewer report |
| **KL-F4** | 報酬再計算スナップショット未実装（最新の取り分レートでのみ再計算可、過去レートの遡及再現は不可） | 過去受注の遡及計算 | Sprint 5 Reviewer report |
| **KL-EMP** | ログインユーザーが staff レコード未紐付け → コンソールに `[UiPrefs] /staff/me 取得失敗、デフォルト値で動作` 警告 | UI 設定 / 権限制御の一部 | 2026-05-16 実機確認 |
| **KL-MIG005** | 本番 tenant_004 で migration 005 全体が未適用。本日 2026-05-27 に orders 列 8 件のみ緊急追加適用済。`invoices` / `quotes` / `invoice_items` / `quote_items` / `shipping_zones` / `shipping_rates` テーブル自体が未作成 | 請求機能 / 発送料計算 — 本番運用で初めて触ると 500 になる可能性 | ADR-034 rev.2、[[project-tenant-review]] 2026-05-27 追記 |
| **KL-PHASE-B** | Phase B は v1.3 で in-scope だが、本番フラグ初期値は要しんごさん確認 | F6 承認時の `products.stock_quantity` 実更新フロー | spec v1.3 F9 |
| **KL-MOCK** | Playwright E2E は API mock + Firebase Auth bypass | E2E に PASS と書かれていても実 API / 実 DB 経路は別途検証要 | CLAUDE.md §ADR-038 |
| **KL-C2** | 過去 634 件の売上データが未移行 | レポート / ダッシュボードの歴史データが少ない可能性 | `ADR-021_completion_report.md` §C2 |
